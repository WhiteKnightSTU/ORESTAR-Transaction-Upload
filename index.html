ORESTAR Transaction Export

Pulls Receipts and Payments from Manager that haven't been exported yet, and builds an ORESTAR-compliant XML file.
This generates a file for manual upload via ORESTAR's "Upload File" feature. It does not file anything automatically — the candidate or treasurer must still log into ORESTAR, review the uploaded transactions, and file/attest to them there.
Configuration
One-time setup — paste each custom field's GUID (Settings > Custom Fields > edit the field > copy the ID from the URL).
Click "Load Accounts" below.

1. Review Transactions

Type/Sub-type are read from your shared custom field. Fix anything it didn't capture (a description for "Miscellaneous" categories, a payment method/check number, etc).

2. Confirm Contacts

ORESTAR requires each contact classified as Individual/Business/Committee/etc., with a name broken into the right fields. If a contributor's or payee's total for the year is over $100, ORESTAR also expects address (and, for individuals, occupation and employer) — add those here if you have them.

3. Generate File

Do this only after you've actually saved/uploaded the file — it's what keeps these transactions out of your next export.
